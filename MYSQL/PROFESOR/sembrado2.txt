**Notas importantes sobre este script:**

1.  **`USE catalogo_profesor_bd;`**: He añadido esta línea para asegurarme de que las inserciones se realicen en la base de datos correcta, según la definición que proporcionaste.
2.  **`AUTO_INCREMENT` y `id`**: Tus tablas están definidas con `AUTO_INCREMENT` para la columna `id`. Sin embargo, en un `INSERT` puedes especificar el `id` manualmente si tienes un valor específico que deseas usar (como hice en este script para mantener un orden y claridad en los ejemplos). Si omites la columna `id` de la lista de columnas en el `INSERT`, MySQL asignará automáticamente el siguiente valor. Para este script, los he especificado para facilitar la referencia en las claves foráneas.
3.  **Contraseñas (`clave`)**: Tal como mencioné antes, he utilizado `SHA2('tu_contraseña', 256)` para hashear las contraseñas. **Esto es una buena práctica de seguridad.** Nunca uses contraseñas en texto plano en una base de datos real.
4.  **`fecha_hora`**: La columna `fecha_hora` en `calificaciones` tiene un `DEFAULT current_timestamp()`. Esto significa que si no la especificas en el `INSERT`, se usará la fecha y hora actuales. En el script, las he especificado explícitamente para los ejemplos.
5.  **Orden de inserción**: El script está diseñado para insertar datos en un orden que respete las dependencias de claves foráneas:
      * `roles` (no depende de nadie)
      * `usuarios` (depende de `roles`)
      * `categorias` (no depende de nadie)
      * `productos` (depende de `categorias`)
      * `calificaciones` (depende de `usuarios` y `productos`)
      * `imagenes` (depende de `productos`)
      * `etiquetas` (depende de `productos`)
6.  **`SET FOREIGN_KEY_CHECKS = 0;` / `SET FOREIGN_KEY_CHECKS = 1;`**: Estas líneas son cruciales. Temporalmente desactivan las comprobaciones de claves foráneas, lo que permite insertar datos en tablas "hijas" antes que en sus tablas "padre" si fuera necesario, sin que MySQL arroje errores. Luego se reactivan.
