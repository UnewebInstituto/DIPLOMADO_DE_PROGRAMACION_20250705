As a programmer, here's a summary of common notations for declaring variables.

There isn't a single universal standard; instead, different programming languages and development teams use various conventions to improve code readability and consistency. These conventions are often referred to as **naming conventions** or **notations**.

## Tipos de Notaci√≥n

### **1. Camel Case (camelCase)** üê™
This is a popular style where the first letter of the first word is lowercase, and the first letter of every subsequent word is capitalized. It's widely used in languages like **Java**, **JavaScript**, and **Swift**.
* **Ejemplo**: `miNombreDeVariable`, `numeroDeEstudiantes`, `obtenerDatosDeUsuario`

---

### **2. Pascal Case (PascalCase)**
Similar to camel case, but the first letter of *every* word is capitalized. This is commonly used for class names, constructors, and sometimes for function names in languages like **C#**, **Python** (for classes), and **Java**.
* **Ejemplo**: `MiClaseDeUsuario`, `NumeroTotalDeProductos`, `NombreDeJugador`

---

### **3. Snake Case (snake_case)** üêç
In this notation, all letters are lowercase, and words are separated by underscores (`_`). It's the standard for variable and function names in languages like **Python** and **Ruby**.
* **Ejemplo**: `mi_nombre_de_variable`, `numero_de_estudiantes`, `obtener_datos_de_usuario`

---

### **4. Kebab Case (kebab-case)**
This notation uses hyphens (`-`) to separate words. It's often used in **CSS** for class and ID names and in file names, but it is **not** a valid notation for variables in most common programming languages (as hyphens are typically interpreted as the subtraction operator).
* **Ejemplo**: `mi-clase-de-estilo`, `nombre-de-archivo`

---

### **5. Notaci√≥n H√∫ngara (Hungarian Notation)**
This is a more classic style where a prefix is added to the variable name to indicate its data type. It was once very popular, particularly in **C++** and **Windows API programming**, but its use has declined with the rise of modern IDEs that can easily show a variable's type.
* **Ejemplo**: `strNombre` (where `str` indicates a string), `intNumeroDeProductos` (where `int` indicates an integer), `bEstaActivo` (where `b` indicates a boolean).
